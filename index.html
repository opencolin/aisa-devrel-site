THE SETUP

Real product. No developer brand. Broken front door.

AISA.one already ships the hard part: one API key that fronts a 9-provider model gateway plus ~650 data and action endpoints across 14 families, plus 41 packaged skills, with machine-to-machine payments wired in behind it. The substrate is real and unusually deep. What is missing is everything that turns a substrate into an ecosystem.

There is no visible DevRel motion, no community loop, and the flagship /models page renders "0 models" because the tRPC and pricing fetch 404s. The site claims "5,000+ agents" but there are ~0 named public builders anyone can point to. That is the gap: not more surface area, but a working front door and a reason for builders to stay. Fix the funnel before flooding it. Kill the vanity numbers, unbreak the page, ship one honest 10-minute path to a real agent, then measure the only thing that matters.

THE REAL WEDGE

Three products that are secretly one substrate

Cheap model routing is a commodity. OpenRouter and Vercel AI Gateway already won the "same base_url, lower token price" game, and the margins prove it. AISA is not really in that fight. The gateway is table stakes. What sits behind the same key is the actual product: ~650 data and action endpoints across 14 families, 41 packaged skills, and machine-to-machine payments. An autonomous agent needs a model, live data, an action to take, and a way to pay for it. AISA is the only one of the three that puts all four behind one key and one wallet.

That is the wedge. A gateway routes tokens. AISA routes capabilities plus settlement. The moment an agent makes a model call *and* pulls an Apollo lead list *and* enriches it with DataForSEO in the same session, switching cost stops being a base_url swap and becomes a rewrite of the agent's entire capability graph. The north-star metric is built on exactly this behavior: Weekly Active Agents making multi-capability calls, payment-weighted once x402 is GA. Not signups. Not followers.

The honest caveat up front: the payment layer (x402, Circle Nanopayments, MPP) is Private Beta, and Foundry is Coming Soon. So today the moat is capabilities-beyond-tokens and agent-native distribution. The payments moat goes public later, on purpose.

MOAT 01

Capabilities, not tokens

A gateway sells you a cheaper model. AISA sells you what the agent does with it: 445 DataForSEO endpoints, 54 Apollo, 46 AgentMail, 32 Twitter, plus Financial, CoinGecko, Polymarket, Kalshi. That catalog is the reason to switch, and it has no equivalent on OpenRouter or Vercel AI Gateway.

MOAT 02

One key, one wallet

9-provider model gateway + 650 data/action APIs + M2M payments behind a single credential. Competitors give you a routing key and stop there. Every additional capability an agent wires up deepens lock-in, because the alternative is stitching a dozen vendor accounts and billing relationships by hand.

MOAT 03

Autonomous settlement

x402 / HTTP-402, Circle Nanopayments, and MPP let an agent pay per call without a human in the loop. This is the part no AI gateway has. It is Private Beta today, which is why payments GTM is sequenced last, but it is what turns a capability network into a transaction network.

MOAT 04

Agent-native distribution

llms.txt, per-page .md doc mirrors, an A2A agent card, and one-prompt onboarding mean an agent can discover and integrate AISA without a human reading docs. The buyer is the agent. That is a distribution surface a token router built for humans does not have.

THE BUG / DAY-ONE FIX

/models renders "0 models"

The flagship discovery page is a P0. The tRPC call plus pricing fetch 404s and the grid falls back to zero, so the first thing a builder sees on a 9-provider gateway is an empty catalog. This is the top of the funnel and it's dark. v0.1 unbreaks it, kills the "1000+ LLMs" claim for one honest count, and instruments the page so we know what conversion actually is. Nothing gets flooded before the front door works.

THE POSITIONING INSIGHT

The wedge isn't cheap tokens

Model routing is a commodity; competing with OpenRouter and Vercel AI Gateway on price is a losing seat. What's actually rare is what sits behind the same key: ~650 data/action endpoints across 14 families plus autonomous M2M payments. The three products aren't three products, they're one substrate. The 10-minute flagship should prove it by wiring a model route to Apollo plus DataForSEO in a single agent. That's the demo nobody else can run.

THE EDGE

Depth the catalog already has

DataForSEO at 445 endpoints, Apollo at 54, plus Financial, CoinGecko, Polymarket, and Kalshi is an unusually deep bench for quant/fintech and GTM-automation builders. That's a real, present advantage, not a roadmap promise. The gap is distribution, not surface area: "5,000+ agents" claimed with roughly zero named public builders. Turning that catalog into named, shipping agents is the entire opportunity.

THE INITIATIVES

Six programs, run in sequence. Each one attacks a specific lever.

These are not parallel bets. They ship in roadmap order because the order is the strategy: fix the funnel before flooding it. You cannot run a launch moment on top of a /models page that renders "0 models," and you cannot sell payments GTM before M2M is near-GA. Every initiative maps to one arc and one lever, and every one moves toward the same north star: Weekly Active Agents making multi-capability calls (a model route plus at least one data-API family in the same period), payment-weighted once x402 is GA. Not signups, not followers.

Each card below states what it fixes, what ships, and how we know it worked. Where the product loses today (broken flagship page, payments not GA), we say so and sequence around it rather than paper over it.

SWOT / MARKET POSITION

Where AISA wins, where it bleeds, and what's actually contestable

The honest read against OpenRouter and Vercel AI Gateway. Both of those are excellent at cheap, fast model routing, and that is a commodity race AISA should not run. The defensible ground is everything past the token: ~650 data and action endpoints across 14 families, autonomous M2M payments, and agent-native distribution, all behind one key and one wallet. The catch is that the front door is broken today and there are ~0 named public builders, so the strengths are latent until the funnel is fixed.

Read each quadrant as a claim we would stand behind in public, including the parts where AISA loses right now.

STRENGTHS

Capabilities beyond tokens, one key

Real surface area competitors do not have: 9-provider OpenAI-compatible gateway PLUS ~650 data/action endpoints across 14 families (DataForSEO 445, Apollo 54, AgentMail 46, Twitter 32, Financial 22) PLUS 41 packaged skills. Gateway, data APIs, and skills are secretly one substrate.

STRENGTHS

Payments as a moat, not a feature

Machine-to-machine settlement via x402/HTTP-402, Circle Nanopayments, and MPP. OpenRouter and Vercel AI Gateway route tokens; neither lets an agent pay for arbitrary capabilities autonomously. This is the part of the story a competitor cannot copy in a sprint.

STRENGTHS

Agent-native by construction

llms.txt, per-page .md doc mirrors, an A2A agent card, and one-prompt onboarding. The product is legible to the agents that are meant to consume it, not just to humans reading marketing pages.

STRENGTHS

Day-one distribution channel

Seller brings an ~86k-developer community (dabl.club, 36-40% open rate) as an attributed acquisition channel, plus DevRel leadership from Cisco Meraki and MetaMask. Distribution is usually the hardest DevRel cold-start; here it exists on signing.

WEAKNESSES

Broken flagship /models page

The /models page renders '0 models' (tRPC + pricing fetch 404/fail). The single highest-intent page for an evaluating builder is dead. This is P0: unbreak it before any traffic is driven anywhere.

WEAKNESSES

No named public builders

'5,000+ agents' is claimed but there are ~0 named public builders and no showcase. The proof layer that turns a curious dev into a committed one does not exist yet.

WEAKNESSES

No DevRel motion, no community loop

No visible developer relations activity, no Discord plumbing, no feedback loop. Real surface area with no one narrating it, staffing it, or closing the loop with the people using it.

WEAKNESSES

Credibility-eroding numbers

Inflated claims (e.g. '1000+ LLMs') undercut trust with exactly the skeptical builder audience AISA needs. One honest number beats ten aspirational ones for this crowd.

OPPORTUNITIES

'Cheap model routing' is a commodity OpenRouter and Vercel already win. Capabilities-beyond-tokens + autonomous payments is uncontested. A 10-min 'build a GTM agent' tutorial (model + Apollo + DataForSEO) demonstrates the wedge in one sitting.

Unusually deep verticals
